## 二、核心体验问题

### 🔴 P0 - 严重影响使用

#### 1. 状态栏占用空间过大
**问题描述**：当前使用 5 个独立的 NSStatusItem，占用菜单栏约 200px 宽度，在 MacBook 屏幕上过于拥挤。

**用户影响**：
- 与其他菜单栏应用冲突
- 小屏幕设备体验极差
- 不符合 macOS 菜单栏应用的主流设计

**竞品参考**：
- iStat Menus：支持合并显示模式
- Stats：单图标 + 下拉面板
- RunCat：单图标动态显示

**建议方案**：
```
方案 A：单图标模式（推荐）
- 一个状态栏图标，显示最关键指标（如 CPU%）
- 点击展开下拉面板显示所有信息

方案 B：可折叠模式
- 默认显示 1-2 个核心指标
- 其他指标收纳到下拉菜单

方案 C：自定义布局
- 允许用户选择显示哪些图标
- 支持拖拽排序
```

#### 2. 缺少应用图标
**问题描述**：assets 目录为空，无 AppIcon.icns，应用在 Finder/Launchpad 显示为默认图标。

**用户影响**：
- 缺乏品牌识别度
- 显得不专业、未完成
- 影响用户信任感

**建议方案**：
- 设计简洁的监控主题图标（建议使用仪表盘/波形/芯片元素）
- 提供 @1x/@2x/@3x 多分辨率版本

---

### 🟠 P1 - 影响用户体验

#### 3. 信息展示密度问题
**问题描述**：
- 菜单栏文字两行显示，部分信息可读性差
- 网络模块使用 ▲▼ 符号，视觉上略显突兀
- 温度显示格式 "CPU\n45C" 不够直观

**建议方案**：
- 优化排版，考虑使用图标代替部分文字标签
- 网络速度使用更直观的上下箭头图标
- 温度显示可考虑带颜色编码（正常/警告/危险）

#### 4. 图表窗口功能有限
**问题描述**：
- 仅显示温度历史，无 CPU/内存/网络历史
- 固定显示 CPU/GPU/SSD，无法切换
- 窗口大小固定，无法调整
- 无时间轴标注

**建议方案**：
- 增加更多图表类型（CPU、内存、网络流量）
- 支持用户自定义监控项
- 添加时间轴、缩放功能
- 支持窗口大小调整

#### 5. 缺少通知/告警功能
**问题描述**：无法在指标异常时通知用户。

**用户场景**：
- CPU 温度过高时预警
- 磁盘空间不足时提醒
- 内存占用过高时通知

**建议方案**：
- 添加阈值设置功能
- 支持 macOS 原生通知
- 可配置告警级别和方式

---

### 🟡 P2 - 优化建议

#### 6. 温度传感器兼容性
**问题描述**：
- 依赖 SMC 传感器，不同 Mac 型号传感器名称不同
- Apple Silicon 和 Intel Mac 处理逻辑不同
- fallback 使用 powermetrics 需要权限

**建议方案**：
- 增加传感器自检测和适配
- 提供传感器选择界面
- 首次运行时引导授权

#### 7. 缺少偏好设置界面
**问题描述**：所有设置通过菜单完成，无独立的偏好设置窗口。

**缺失功能**：
- 开机自启动设置
- 显示项目自定义
- 主题/颜色设置
- 告警阈值设置

#### 8. 无国际化支持
**问题描述**：界面文案硬编码为英文，无多语言支持。

**建议方案**：
- 添加中文界面支持
- 使用 Rust 国际化库（如 fluent）

#### 9. 缺少数据导出功能
**问题描述**：无法导出历史数据用于分析。

**建议方案**：
- 支持 CSV/JSON 导出
- 可选的时间范围选择

## 六、总结

**核心优势**：
- Rust 实现，性能优秀，资源占用低
- 开源免费，可定制性强
- 基础功能完整

**核心短板**：
- 状态栏占用过大（最需解决）
- 缺少应用图标和品牌形象
- 缺少告警和自启动等实用功能

**建议**：优先解决 P0 问题（状态栏占用、应用图标），再逐步完善功能和体验。以「轻量、美观、实用」为产品定位，与 iStat Menus 的功能全面形成差异化竞争。
